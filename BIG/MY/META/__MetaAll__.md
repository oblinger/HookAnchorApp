### --- Notes On My Filing System ---

EVERYTHING -- **ALL** of Dan's Digital info fits into exactly one of these three segments.
- Stored in first appropriate segment:  (1) specialized, (2) by name, (3) in stream
  priority within STREAM:  processing, speciality, evernote/WP

- SPECIALIZED  -- Certain info is best managed using specialized tools and structures.  Including:
    -- EXTERNAL   Phone/SMS/Email, Calendar, Contacts, Fin&Acct Info,
    -- OTHER      Music, Photos, ... Flat (Mobile accessible)
- BY NAME      -- Certain info is densely clustered around a given 'named' topic.
                  In this case it is under a folder with that topic name  (single word/abbref preferred)
    -- PROJ    -- Code or substanative projects
    -- TOPICS  -- Named topics (often in /ob/ref/topics)  ALWAYS accessible by CAPS-J NAME or CAPS-L NAME
- IN STREAM    -- Info not above is captured in exactly ONE stream
  ARCHIVES     -- (bin,proj,ref,WP) are archives of corrsponding folders.
  EVERNOTE
    -- PHONE   -- Short, Transactional, Buisness Related, While On Phone.
    -- SNIP    -- 'Snipits' written by me, but not yet in document/email/etc.
                  Offloaded notester notes etc. are tagged archived in this notebook
    -- REFS    -- Ref material for posterity:  snipit not by me, web searches, docs, etc.
  LOGS
    -- SPECIAL -- Bought, Ext, Humor, Ideas, Letters
    -- PROC    -- These are 'holding' logs for subsequent processing.  Often called 'To...'
